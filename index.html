Что такое верстка?

Процесс компоновки элементов веб-страниц посредством HTML и стилизации с помощью CSS. Чаще по графическому макету.

В теории довольно просто - смотрим в макете цвета, шрифты, замеряем отступы, нарезаем картинки. Переносим в HTML/CSS.

Но есть несколько моментов.

Почему не 100%-ное соответствие макету

  • Средство создания макета (Illustrator, Photoshop и др.) и браузер могут отображать одно и тоже по-разному.
  • Разные браузеры
  • Один браузер, разные ОС и разные устройствах
  • Один браузер, разные версии
  • Один браузер, одна ОС, разное "железо"

  • производительность

Graceful Degradation

общетехнический теримн

Сайт отображается хорошо в новых браузерах, в старых - могут быть отличия в визуальном отображении, удобстве использования, но при этом основные функции выполняются.

пример: логинация на сайте - красивое оформление, ajax-обработка

Как взаимодействовать дизайнерам и верстальщикам (технологам, разработчикам)

Веб-дизайнер — это инженер. Веб-дизайнер — это не рисовальщик, он не придумывает «как будет выглядеть», он придумывает «как сделать, чтобы проект лучше всего решал поставленные задачи» и как потом воплотить это визуально. Отрисовка макета без представления о том, как будет работать сайт, напоминает проектирование автомобиля без представления о его двигателе, системах управления, условиях эксплуатации, количестве и расположение конечностей водителя.
Николай Громов, HTMLAcademy
Между дизайнером в одном углу и технологом в другом — не пропасть, между ними градиент. Расширить диапазон ваших знаний на этом градиенте поможет знание веб-технологий, базовых инструментов и приёмов, родных для веба. Полиграфист вы, шрифтовик или пишете маслом, веб — ваша витрина, его возможности важно хорошо знать.
Вадим Макеев, евангелист Opera Software
Навык вёрстки у дизайнера позволяет, с одной стороны, избежать решений, которые противоречат логике веб-дизайна и которые невозможно воплотить в жизнь, а с другой — использовать на полную катушку возможности современных технологий вёрстки, сделать сайт более элегантным, интересным, динамичным.

Ошибки дизайнеров

Макет нужно делать в цветовом профиле sRGB (sRGB IEC61966-2.1)

Иначе цвета в вёрстке могут отличаться от макетных.

Используйте сетку

Сетка для структуры блоков

Вертикальный ритм для текста

Структура слоев

Группируйте слои и придерживайтесь иерархии в макете. Давайте слоям осмысленные названия.

В макетах допустим только режим наложения normal

Как для слоев, так и для эффектов. Так будет, пока не появится кроссбраузерная поддержка режимов наложения. И даже когда появится, использование режимов наложения нужно будет применять осторожно, с огладкой на особенности реализации в вебе. Это правило не касается состоящих из множества слоёв иллюстраций, которые планируется соединять в одно изображение.

Оставляйте эффекты слоёв

Эффекты, преобразованные в слои, ощутимо дольше верстать. Используйте Layer Styles слоев вместо растированных слоев.

Избегайте менять кернинг и трекинг

На подавляющем большинстве дисплеев очень мало «пикселей на дюйм», такое изменение нередко приводит к грустным эффектам: размытие символов, изменение их визуальной плотности. Некроссбраузерное отображение для некоторых шрифтов.

Изменение цвета объектов или текста при помощи прозрачности

Плохо, так как технологически мы стараемся избегать прозрачности — много полупрозрачных объектов могут вызвать проблемы быстродействия страницы.

Набор текстовых блоков КАПСОМ вместо использования «All Caps»

Приводит к необходимости нажимать несколько лишних кнопок при копировании текста слоя. Когда таких слоёв много — потеря времени.

Кастомные шрифты

  • Не используйте большие количество несистемных шрифтов (не больше 2-3). Проблемы мерцания и задержки отображения текста во время загрузки.
  • Используйте, по возможности, бесплатные шрифты (google.com/fonts).
  • А если использовать системные? "Безопасные" веб-шрифты

В вебе нет кроссбраузерных переносов

Лучше не использовать переносы слов вообще. Расстановка символов мягкого переноса в тексте возможна (никто не будет делать это вручную, а автоматическое ещё нужно внедрить в проект), автоматический перенос в браузерах имеет скромную поддержку.

По возможности, не стилизовать стандартные контролы.

Некроссбраузерный вид, но высокая доступность.

  • Выпадающие списки. Блеск и нищета стандартных селектов
  • Плейсхолдеры в полях формы
  • Всплывабщие подсказки
  • Скролл-бары

Для веба рисуйте векторные формы вместо работы кистью (например, вы используете планшет).

Заливки и формы, созданные кистью получаются «грязными» — имеют множество лишних ключевых точек, что крайне негативно сказывается на размере файла.

Выравнивайте точки в кривых по пиксельной сетке

Тогда иконки не будут мыльными на экранах с низким разрешением, к тому же будут лучше сжиматься. В редакторах настройка обычно называется «snap to pixel».

Артборд должен быть подогнан по габаритам фигуры.

Для нескольких иконок одинакового размера допустимо, чтобы атрборды были одинаковых размеров (больше габарита фигур).

Предложения по организации работы

Файлы

  • Для PSD-файлов: одна страница - один файл (по соображениям производительности). Допускается, отдельные элементы (модальные окна, выпадающие блоки) помещать в этот файл.
  • Для AI-файлов: одна страница - один artboard или слой.
  • Хранение версий файлов, например, main.v1.ai, main.v2.ai и т.д. Файл changelog.txt для описания изменений.
  • Группировка элементов в блоки
  • Cемантические названия слоев

Ресурсы

Прикладвать все прилинкованные ресурсы (картинки, шрифты, паттерны заливки)

Макеты для резиновых и адаптивных сайтов

  • Для резиновых сайтов желательно иметь один-два макета: узкий (например, 1024px) и/или один всегда 1920px.
  • Не забывайте, что область просмотра браузера не равна разрешению экрана. Можно использовать mockup c изображением браузера для упрощения работы.

Адаптивность

  • Выбор 2-3 брейкпойнтов.
  • Bootstrap-брейкпойнты: 768, 992, 1200 пикселей. Можно добавить еще 320 и 480.
  • Сделав макет в ширине 768, Вы показали то, как макет выглядит от 768 до следующего брейкпойнта в большую сторону (скажем, 992).
  • Подход: контент должен определять breakpoint'ы.
  • Требования к макетам для отзывчивого дизайна

Разработка UI Kit и styleguide. Брэд Фрост. Атомарный веб-дизайн

Содержание UI Kit

  • Сетка
  • Типографика. Цвет, заголовки, семейства шрифтов, вертикальный ритм текста.
  • Контролы и их состояния (normal, hover, active, focus, disabled)
  • Таблицы с данными
  • Набор иконок (лучше отдельный файл, где каждая иконка размещена на отдельном артборде со своим именем)
